- 多用户通讯录系统 – 完整设计方案

  ------

  ### 一、项目概述

  构建一个支持多用户、内置管理员、具备导入/导出/回收站等完整功能的在线通讯录服务。系统采用 **Spring Boot 3.x + Spring Data JPA + Thymeleaf + Bootstrap 5** 技术栈，一套代码同时适配 PC 与移动端，部署为内嵌网页的独立 JAR 包。

  **核心功能清单**

  - 用户注册/登录，基于 Session 的安全认证
  - 系统内置唯一管理员账号，可管理所有用户（增、删、改、禁用）
  - 分组管理（默认分组受保护，删除分组时联系人自动归入默认分组）
  - 联系人信息全面（姓名、性别、多类型电话、邮箱、单位、职位、省/市/区、详细地址、生日、备注）
  - 多功能组合查询：按姓名、电话、地域（省/市/县）、单位、职位、分组等筛选
  - 头像上传（后端缩放后存入数据库，预留 CDN 切换能力）
  - Excel 导入/导出通讯录
  - 回收站（软删除，支持恢复、彻底删除、清空）
  - 全局输入校验、权限隔离、事务保障、异常处理

  ### 三、用户角色与权限

  | 角色      | 权限                                                         |
  | :-------- | :----------------------------------------------------------- |
  | **ADMIN** | 唯一内置管理员（账号在配置文件中指定）。可查看所有用户及其联系人、管理用户（增删改禁用）、操作回收站（查看/恢复/删除所有用户联系人） |

  ### 六、核心业务设计

  #### 6.1 用户注册与管理员初始化

  - **管理员初始化**：在 `application.yml` 中配置默认管理员用户名和密码，应用启动时通过 `CommandLineRunner` 检查是否存在该账号，若不存在则自动创建，角色设为 `ADMIN`，且不可被删除或降级。
  - **用户注册**：公开注册表单，提交后校验唯一性，密码 BCrypt 加密，角色固定为 `USER`，同时自动为其创建一个“默认分组”（`is_default=true`），使用 `@Transactional` 保证原子性。
  - **管理员管理用户**：管理员可在 `/admin/users` 页面查看所有用户，执行禁用/启用、修改信息（不能修改角色为 ADMIN）、删除用户（物理删除其所有分组、联系人及回收站数据）。

  #### 6.2 分组管理

  - 每个用户登录后可见自己的分组列表（默认分组排首位）。
  - 新增分组时校验同名冲突。
  - 删除分组：**非默认分组**才可删除。删除前将该分组下所有未被软删除的联系人（`is_deleted=false`）的 `group_id` 更新为当前用户的默认分组 ID，然后删除分组。整个过程在一个事务中完成，若发现是默认分组则抛出业务异常。
  - 默认分组不可删除，前端隐藏其删除按钮，后端二次校验。

  #### 6.3 联系人 CRUD 与软删除

  - 新增联系人：必须提供姓名和至少一个联系电话（手机/家庭/工作至少填一个），若未指定分组则归入默认分组。
  - 编辑联系人：校验归属权（当前用户或管理员）。
  - 删除操作：**前端“删除”执行软删除**，即将 `is_deleted` 置为 `true`，记录 `deleted_at`；在分组删除时仅移动未删除联系人。
  - 回收站：
    - 查看：`GET /contacts/recycle?page=&size=`，仅展示 `is_deleted=true` 的联系人。
    - 恢复：`PUT /contacts/{id}/restore`，将 `is_deleted` 设为 `false`，清除 `deleted_at`，分组恢复为删除前的 `group_id`。
    - 彻底删除：`DELETE /contacts/{id}/permanent`，物理删除记录。
    - 清空回收站：`DELETE /contacts/recycle`，物理删除当前用户所有已删除联系人（管理员可清空任一用户）。

  #### 6.4 多条件查询

  采用 **JPA Specification** 动态组合查询条件，代码清晰且无需引入 MyBatis。

  查询接口 `GET /contacts/data` 接受参数：

  - `keyword`：模糊匹配姓名
  - `phone`：模糊匹配手机、家庭、工作电话
  - `province`, `city`, `district`：精确匹配
  - `company`：模糊匹配
  - `jobTitle`：模糊匹配
  - `groupId`：按分组筛选
  - `page`, `size`：分页

  Service 层构建 Specification，对字符串使用 `%value%`（并转义 `%`、`_` 等通配符），所有查询默认添加 `user_id = :currentUserId AND is_deleted = false`（回收站除外）。

  #### 6.5 头像上传与存储

  - 上传接口：`POST /user/avatar`，接收 `MultipartFile`。
  - 校验：仅允许 `image/jpeg`、`image/png`，大小不超过 2MB。
  - 处理：使用 Thumbnailator 将图片缩放为 **200×200 像素** 的正方形（覆盖裁剪），输出质量 80%，转为字节数组存入 `avatar_data`，同时保存 `avatar_content_type` 和当前时间。
  - 读取：`GET /user/avatar` 或 `GET /user/{id}/avatar`，优先从 `avatar_url` 加载（若不为空），否则从数据库读取 `avatar_data` 返回，设置正确的 `Content-Type` 和缓存头。
  - **CDN 预留**：当未来需要迁移至 CDN 时，可编写一个批处理将 `avatar_data` 上传并更新 `avatar_url`，随后清空 `avatar_data` 字段，代码中仅需调整读取优先级即可。

  #### 6.6 导入导出通讯录

  **导出 Excel**

  - 接口：`GET /contacts/export?{查询参数}`，支持导出当前筛选结果或全部联系人。
  - 使用 Apache POI 的 `XSSFWorkbook`，表头：姓名、性别、手机、家庭电话、工作电话、邮箱、单位、职位、省、市、区、详细地址、生日、备注、所属分组。
  - 性别转为文字，分组显示名称（通过关联查询）。
  - 响应头设置 `Content-Disposition: attachment; filename="contacts.xlsx"`。

  **导入 Excel**

  - 接口：`POST /contacts/import`，接收上传的 Excel 文件。
  - 逐行解析，每行校验：
    - 姓名必填，否则跳过。
    - 分组处理：如果填写了分组名，先查找该用户下是否存在，不存在则自动创建；未填写则归入默认分组。
    - 数据格式校验：手机号正则（可选）、邮箱格式、日期格式等。
    - 根据姓名+手机号判断重复（可配置跳过或更新，默认跳过）。
  - 记录成功/失败行及原因，返回 `ImportResult` DTO（成功条数、失败详情列表）。
  - 限制文件大小（如 10MB）和最大行数（如 5000 行），防止 OOM。

  #### 6.7 校验与鲁棒性

  - **Bean Validation**：所有 DTO 标注 `@NotBlank`、`@Size`、`@Email` 等，自定义校验器确保“至少一个联系电话”。
  - **全局异常处理**：`@ControllerAdvice` 捕获 `MethodArgumentNotValidException`、`BusinessException`、`AccessDeniedException` 等，返回统一 JSON 或错误页面。
  - **事务**：涉及多表操作（注册、分组删除、导入）均使用 `@Transactional`。
  - **权限隔离**：Service 层通过 `SecurityContextHolder` 获取当前用户，所有查询均附加 `user_id` 条件；管理员可通过管理员接口跨用户操作。
  - **防 XSS/注入**：Thymeleaf 默认转义，JPA 参数化查询。
  - **并发安全**：删除分组时，使用悲观锁或版本号，保证联系人移动不发生丢失（本系统并发量低，依赖数据库事务即可）。

  ### 八、安全设计

  - 使用 Spring Security 的 `HttpSecurity` 配置：
    - 登录页、注册页、静态资源允许匿名访问。
    - 其他所有请求需认证。
    - 启用 CSRF，Thymeleaf 模板自动插入 `_csrf`，AJAX 请求从 meta 读取 token。
  - 密码编码器：`BCryptPasswordEncoder`。
  - 管理员账号初始化后禁止注册同名账号（唯一约束）。
  - Session 超时 30 分钟，超时后重定向登录页。
