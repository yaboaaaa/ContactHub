# 第3章 详细设计

## 3.1 数据库设计

### 数据库结构设计

数据库使用 MySQL 8.0+，通过 JPA `ddl-auto: update` 自动建表，无需手动编写 SQL 迁移脚本。所有时间戳使用 `LocalDateTime`。

#### 用户信息表（users）

| 字段名称 | 数据类型 | 约束 | 备注 |
| :--- | :--- | :--- | :--- |
| id | BIGINT | PK, AUTO_INCREMENT | 用户编号，主键自增 |
| username | VARCHAR(50) | UNIQUE, NOT NULL | 登录用户名，全局唯一 |
| password | VARCHAR(200) | NOT NULL | 密码，BCrypt 加密后的密文 |
| email | VARCHAR(100) | | 用户邮箱，可选填写 |
| nickname | VARCHAR(50) | | 昵称，未填写时系统自动生成随机昵称 |
| role | VARCHAR(20) | NOT NULL, DEFAULT 'USER' | 用户角色：USER（普通用户）或 ADMIN（管理员） |
| enabled | BOOLEAN | NOT NULL, DEFAULT TRUE | 账号状态：true=启用，false=禁用 |
| avatar_url | VARCHAR(500) | | 用户头像 URL 地址，指向 uploads/avatars/ 目录 |
| created_at | DATETIME | NOT NULL | 账号创建时间，JPA @PrePersist 自动填充 |
| updated_at | DATETIME | | 最后更新时间，JPA @PreUpdate 自动填充 |

索引：`username` 字段建立唯一索引，保证用户名全局唯一。

#### 联系人分组表（contact_groups）

| 字段名称 | 数据类型 | 约束 | 备注 |
| :--- | :--- | :--- | :--- |
| id | BIGINT | PK, AUTO_INCREMENT | 分组编号，主键自增 |
| name | VARCHAR(50) | NOT NULL | 分组名称 |
| user_id | BIGINT | FK -> users.id, NOT NULL | 所属用户 ID，外键关联 users 表 |
| is_default | BOOLEAN | NOT NULL, DEFAULT FALSE | 是否为默认分组，每个用户有且仅有一个默认分组 |
| sort_order | INTEGER | DEFAULT 0 | 排序顺序，数值越小越靠前，默认分组始终排首位 |
| created_at | DATETIME | NOT NULL | 分组创建时间 |

#### 联系人信息表（contacts）

| 字段名称 | 数据类型 | 约束 | 备注 |
| :--- | :--- | :--- | :--- |
| id | BIGINT | PK, AUTO_INCREMENT | 联系人编号，主键自增 |
| user_id | BIGINT | FK -> users.id, NOT NULL | 所属用户 ID，用于多用户数据隔离 |
| group_id | BIGINT | FK -> contact_groups.id, NOT NULL | 所属分组 ID |
| name | VARCHAR(50) | NOT NULL | 显示名（完整姓名），由 family_name + given_name 拼接 |
| family_name | VARCHAR(30) | | 姓，符合 CardDAV 标准 |
| given_name | VARCHAR(30) | | 名，符合 CardDAV 标准 |
| uid | VARCHAR(255) | UNIQUE | CardDAV 兼容唯一标识，系统自动生成 UUID |
| gender | INTEGER | | 性别：0-不便透露，1-男，2-女 |
| phone_mobile | VARCHAR(20) | | 手机号码 |
| phone_home | VARCHAR(20) | | 家庭电话 |
| phone_work | VARCHAR(20) | | 工作电话 |
| email | VARCHAR(100) | | 电子邮箱 |
| company | VARCHAR(100) | | 工作单位 |
| job_title | VARCHAR(50) | | 职位 |
| province | VARCHAR(30) | | 省份 |
| city | VARCHAR(30) | | 城市 |
| district | VARCHAR(30) | | 区/县 |
| address_detail | VARCHAR(200) | | 详细地址 |
| birthday | DATE | | 生日 |
| notes | VARCHAR(500) | | 备注 |
| avatar_url | VARCHAR(500) | | 联系人头像 URL 地址 |
| is_deleted | BOOLEAN | NOT NULL, DEFAULT FALSE | 软删除标记：false=正常，true=已删除 |
| deleted_at | DATETIME | | 删除时间，软删除时记录，恢复时清除 |
| created_at | DATETIME | NOT NULL | 创建时间 |
| updated_at | DATETIME | | 最后更新时间 |

索引：为 `user_id`、`group_id`、`is_deleted`、`name`、`phone_mobile` 建立单字段索引；建立组合索引 `(province, city, district)` 加速省市区三级地域查询。
